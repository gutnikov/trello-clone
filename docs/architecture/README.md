# Architecture

Architecture Decision Records (ADRs) and design documentation.

## What Goes Here

- **ADRs** — Records of significant architectural decisions (use `adr-template.md`)
- **Design docs** — System design and component interaction diagrams

## Naming Convention

ADRs: `adr-001-short-title.md`, `adr-002-short-title.md`
Design docs: `design-component-name.md`

## Creating a New ADR

1. Copy `adr-template.md`
2. Number it sequentially (check existing ADRs for the next number)
3. Fill in Context, Decision, Consequences, and Alternatives
4. Set status to "Proposed" until accepted by the team
